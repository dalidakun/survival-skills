# 查找大文件的方法

## 问题
GitHub Desktop 提示超过 100MB，但您看到的只有 40MB。

## 可能的原因

1. **有隐藏的大文件**（如 `.android`、`.gradle` 等）
2. **多个文件加起来超过 100MB**
3. **PDF 文件太大**
4. **Git 历史记录中包含大文件**

## 解决方法

### 方法 1：在 GitHub Desktop 中查看所有文件

1. **在 GitHub Desktop 左侧文件列表中**
2. **查看每个文件的大小**（如果有显示）
3. **特别检查这些文件夹**：
   - `.android/` - Android 模拟器文件
   - `.gradle/` - Gradle 缓存
   - `AppData/` - 应用数据
   - `public/pdfs/` - PDF 文件
   - `node_modules/` - 依赖包（应该不显示）

### 方法 2：使用 Windows 文件管理器查找大文件

1. **打开项目文件夹**
2. **在搜索框中输入**：`size:>50MB`
3. **查看搜索结果**，找出所有大于 50MB 的文件

### 方法 3：检查特定文件夹

在文件管理器中检查这些位置是否有大文件：

- `C:\Users\Administrator\Desktop\生存技能\.android\`
- `C:\Users\Administrator\Desktop\生存技能\.gradle\`
- `C:\Users\Administrator\Desktop\生存技能\AppData\`
- `C:\Users\Administrator\Desktop\生存技能\public\pdfs\`

### 方法 4：在 GitHub Desktop 中取消选择所有大文件

1. **在文件列表中，取消勾选**：
   - 所有 `.android` 相关文件
   - 所有 `.gradle` 相关文件
   - 所有 `AppData` 相关文件
   - `public/pdfs/` 中的大 PDF 文件（如果有）

2. **只勾选必要的项目文件**：
   - `src/`
   - `content/`
   - `public/`（但排除 `pdfs/` 中的大文件）
   - `package.json`
   - `package-lock.json`
   - 配置文件

## 快速检查清单

在 GitHub Desktop 中，确保这些文件/文件夹**没有被勾选**：

- [ ] `.android/` 文件夹
- [ ] `.gradle/` 文件夹  
- [ ] `AppData/` 文件夹
- [ ] `node_modules/` 文件夹（应该不显示）
- [ ] `public/pdfs/` 中的大 PDF 文件
- [ ] 任何 `.jar` 文件
- [ ] 任何 `.dll` 文件
- [ ] 任何 `.img` 文件
- [ ] 任何 `.qcow2` 文件

## 如果还是提示超过 100MB

可能是 Git 历史记录中已经包含了大文件。需要清理历史记录（这需要一些技术操作）。
