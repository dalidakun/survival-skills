# 解决 GitHub 大文件问题

## 问题
GitHub Desktop 提示有些文件超过 100MB，无法上传。

## 原因
这些文件是 Android 开发相关的文件（`.android`、`.gradle`、Android SDK），不应该提交到 Git。

## 解决方案

### 方法 1：在 GitHub Desktop 中取消选择大文件（推荐）

1. **在 GitHub Desktop 左侧的文件列表中**
2. **找到这些大文件，取消勾选**：
   - `.android\` 文件夹下的所有文件
   - `.gradle\` 文件夹下的所有文件
   - `AppData\` 文件夹下的所有文件
3. **只勾选项目相关的文件**：
   - `src/` 文件夹
   - `content/` 文件夹
   - `public/` 文件夹（但排除 `pdfs/` 中的大文件）
   - `package.json`
   - `README.md`
   - 其他配置文件

### 方法 2：使用已更新的 .gitignore

我已经更新了 `.gitignore` 文件，添加了排除规则。现在需要：

1. **刷新 GitHub Desktop**
   - 关闭并重新打开 GitHub Desktop
   - 或者点击菜单：**Repository** → **Refresh**

2. **检查文件列表**
   - 大文件应该不再显示在待提交列表中
   - 如果还显示，手动取消勾选

### 方法 3：手动删除这些文件夹（如果不需要）

如果这些 Android 文件不是项目必需的：

1. **在文件管理器中删除**：
   - `.android` 文件夹
   - `.gradle` 文件夹
   - `AppData` 文件夹（如果在项目目录中）

2. **然后重新在 GitHub Desktop 中查看**

## 应该提交的文件

✅ **应该提交的文件**：
- `src/` - 源代码
- `content/` - 文章内容
- `public/` - 静态资源（但排除 `pdfs/` 中的上传文件）
- `package.json` - 依赖配置
- `README.md` - 说明文档
- `next.config.mjs` - Next.js 配置
- `tailwind.config.ts` - Tailwind 配置
- `.gitignore` - Git 忽略规则
- 其他配置文件

❌ **不应该提交的文件**：
- `node_modules/` - 依赖包（已排除）
- `.next/` - 构建文件（已排除）
- `.android/` - Android 模拟器文件（已排除）
- `.gradle/` - Gradle 缓存（已排除）
- `AppData/` - 应用数据（已排除）
- `public/pdfs/` 中的上传文件（已排除）

## 操作步骤

1. **点击 "Cancel" 按钮**（不要点击 "Commit anyway"）

2. **在 GitHub Desktop 中**：
   - 查看左侧文件列表
   - 取消勾选所有大文件（`.android`、`.gradle`、`AppData`）
   - 只勾选项目文件

3. **填写提交信息**：
   - 左下角输入：`Initial commit: 蒙谦自然教育`

4. **点击 "Commit to main"**

5. **如果还有警告，再次检查并取消勾选大文件**

## 验证

- ✅ 没有 `.gradle` 文件夹
- ✅ 没有 `AppData` 文件夹
- ✅ 有 `src/` 文件夹
- ✅ 有 `content/` 文件夹
- ✅ 有 `package.json`
