# 解决 100MB 限制问题的详细步骤

## 问题分析

GitHub Desktop 提示超过 100MB，但您看到的只有 40MB。这是因为：
- 可能有隐藏的大文件（如 `.android`、`.gradle`）
- 多个文件加起来超过 100MB
- PDF 文件可能很大

## 解决步骤

### 步骤 1：在 GitHub Desktop 中检查所有文件

1. **打开 GitHub Desktop**
2. **查看左侧的 "Changes" 标签页**
3. **仔细检查每个文件/文件夹**：
   - 查看是否有 `.android` 文件夹
   - 查看是否有 `.gradle` 文件夹
   - 查看是否有 `AppData` 文件夹
   - 查看 `public/pdfs/` 中的 PDF 文件大小

### 步骤 2：取消选择所有大文件

在 GitHub Desktop 的文件列表中，**取消勾选**以下内容：

#### 必须取消的文件夹/文件：
- [ ] `.android/` - 整个文件夹
- [ ] `.gradle/` - 整个文件夹
- [ ] `AppData/` - 整个文件夹
- [ ] `node_modules/` - 整个文件夹（应该不显示）
- [ ] `public/pdfs/` 中的 PDF 文件（如果很大）

#### 应该保留的文件：
- [x] `src/` - 源代码
- [x] `content/` - 文章内容
- [x] `public/` - 静态资源（但排除 `pdfs/` 中的大文件）
- [x] `package.json`
- [x] `package-lock.json`
- [x] `README.md`
- [x] 所有配置文件（`.gitignore`、`next.config.mjs` 等）

### 步骤 3：如果找不到大文件

如果 GitHub Desktop 中没有显示 `.android` 或 `.gradle` 文件夹，但它们确实存在：

1. **在文件管理器中检查**：
   - 打开项目文件夹
   - 查看是否有隐藏文件夹（需要显示隐藏文件）
   - 在文件管理器中：查看 → 显示 → 隐藏的项目

2. **如果发现这些文件夹**：
   - 它们应该在 `.gitignore` 中（我已经添加了）
   - 刷新 GitHub Desktop
   - 它们应该不再显示

### 步骤 4：处理 PDF 文件

如果 `public/pdfs/` 中有大 PDF 文件：

1. **选项 A：不提交 PDF 文件**（推荐）
   - 取消勾选 `public/pdfs/` 中的 PDF 文件
   - PDF 文件已经在 `.gitignore` 中排除
   - 以后可以通过管理后台上传

2. **选项 B：压缩 PDF 文件**
   - 使用在线工具压缩 PDF
   - 然后替换原文件

### 步骤 5：提交

1. **确认只勾选了必要的文件**
2. **填写提交信息**：`Initial commit: 蒙谦自然教育`
3. **点击 "Commit to main"**
4. **如果还有警告，再次检查并取消勾选大文件**

## 快速检查清单

在点击 "Commit" 之前，确认：

- [ ] 没有勾选 `.android` 文件夹
- [ ] 没有勾选 `.gradle` 文件夹
- [ ] 没有勾选 `AppData` 文件夹
- [ ] 没有勾选 `node_modules` 文件夹
- [ ] 没有勾选 `public/pdfs/` 中的大 PDF 文件
- [ ] 只勾选了项目源代码和配置文件
- [ ] 提交信息已填写

## 如果还是不行

如果按照以上步骤操作后，还是提示超过 100MB：

1. **截图 GitHub Desktop 的文件列表**，我帮您检查
2. **或者告诉我具体哪些文件被勾选了**
