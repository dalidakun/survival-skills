# 检查新文章问题的步骤

## 问题
新文章没有显示在顶端，也没有"更新"标签。

## 检查步骤

### 1. 检查 GitHub 仓库中的文件

访问：https://github.com/dalidakun/survival-skills/tree/main/content

查看最新创建的文件，检查：
- 文件是否已创建
- frontmatter 中是否有 `updatedAt` 字段
- `updatedAt` 字段的格式是否正确（应该有引号）

**正确的格式应该是**：
```yaml
---
slug: xxx
title: xxx
updatedAt: "2024-12-19T10:30:00.000Z"
link: https://...
---
```

### 2. 如果 `updatedAt` 没有引号

如果看到的是：
```yaml
updatedAt: 2024-12-19T10:30:00.000Z
```

这可能导致 YAML 解析错误。我已经修复了代码，新创建的文件会自动加引号。

### 3. 对于已存在的文件

如果旧文件没有引号，可以：
1. 手动编辑 GitHub 上的文件
2. 在 `updatedAt:` 后面添加引号
3. 或者重新提交一次文章（会更新文件）

### 4. 检查 Vercel 日志

在 Vercel 控制台查看：
- Functions → 查看 API 日志
- 是否有错误信息
- 是否有调试输出

## 已修复

我已经修复了代码，现在 `updatedAt` 会自动加引号：
```yaml
updatedAt: "2024-12-19T10:30:00.000Z"
```

这样可以确保 YAML 正确解析。
