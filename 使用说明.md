# 网站使用说明

## 已完成的功能

### 1. 免费部署方案 ✅

网站可以免费部署到 **Vercel**，详细步骤请参考 `DEPLOYMENT.md`。

**快速步骤：**
1. 将代码推送到 GitHub
2. 在 Vercel.com 注册并导入项目
3. 设置环境变量 `ADMIN_PASSWORD`
4. 自动部署完成

### 2. PDF 文件上传和存储 ✅

**功能说明：**
- 管理员可以上传 PDF 文件到服务器
- PDF 文件保存在 `public/pdfs/` 目录
- 上传后自动生成可访问的链接（格式：`/pdfs/文件名.pdf`）
- 支持最大 50MB 的文件

**使用方法：**
1. 登录管理后台
2. 选择"上传 PDF"模式
3. 选择 PDF 文件并上传
4. 系统会自动生成链接并保存到文章列表

**注意事项：**
- 如果使用 Vercel 等平台，上传的文件在重新部署时可能会丢失
- 建议将重要的 PDF 文件提交到 Git 仓库，或使用外部存储服务

### 3. 管理菜单权限控制 ✅

**功能说明：**
- 只有登录的管理员才能看到"管理"菜单
- 普通用户无法看到管理入口
- 使用密码验证保护管理功能

**使用方法：**
1. 设置环境变量 `ADMIN_PASSWORD`（在 `.env.local` 或 Vercel 环境变量中）
2. 访问 `/admin` 页面
3. 输入密码登录
4. 登录后，"管理"菜单会出现在导航栏
5. 退出登录后，菜单会自动隐藏

## 环境变量配置

### 本地开发

创建 `.env.local` 文件：

```
ADMIN_PASSWORD=你的强密码
```

### Vercel 部署

在 Vercel 项目设置中：
1. 进入 Settings → Environment Variables
2. 添加变量：
   - Name: `ADMIN_PASSWORD`
   - Value: 你的强密码
3. 保存并重新部署

## 安全建议

1. **使用强密码**：设置一个复杂的管理员密码
2. **定期更新密码**：定期更换 `ADMIN_PASSWORD`
3. **HTTPS**：Vercel 自动提供 HTTPS，确保传输安全
4. **不要提交敏感信息**：确保 `.env.local` 在 `.gitignore` 中

## 常见问题

### Q: 上传的 PDF 文件在哪里？

A: 文件保存在 `public/pdfs/` 目录，可以通过 `/pdfs/文件名.pdf` 访问。

### Q: 部署后上传的文件会丢失吗？

A: 如果使用 Vercel 等无服务器平台，上传的文件在重新部署时可能会丢失。建议：
- 将重要文件提交到 Git
- 使用外部存储服务（如 GitHub、云存储）
- 使用 Vercel Blob Storage

### Q: 如何让其他用户看不到管理菜单？

A: 管理菜单已经实现了权限控制，只有登录的管理员才能看到。普通用户访问网站时不会看到"管理"链接。

### Q: 如何修改管理员密码？

A: 修改环境变量 `ADMIN_PASSWORD` 的值，然后重新部署（或重启开发服务器）。
