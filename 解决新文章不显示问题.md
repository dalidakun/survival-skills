# 解决新文章不显示问题

## 问题原因

Next.js 默认使用**静态生成（SSG）**，这意味着：
- 首页在构建时就已经生成好了
- 新添加的文章不会自动显示
- 需要重新构建才能看到新内容

## 解决方案

已修改代码，将首页改为**动态渲染**：
- 每次访问首页时，都会重新读取文件
- 新文章会立即显示，无需重新构建

## 修改内容

1. **首页（`src/app/page.tsx`）**：
   - 添加了 `export const dynamic = 'force-dynamic'`
   - 强制动态渲染

2. **文章详情页（`src/app/skills/[slug]/page.tsx`）**：
   - 移除了 `generateStaticParams`
   - 改为动态路由

## 下一步操作

### 1. 提交代码到 GitHub

```bash
git add src/app/page.tsx src/app/skills/[slug]/page.tsx
git commit -m "Fix: 改为动态渲染，确保新文章能立即显示"
git push
```

### 2. 等待 Vercel 自动部署

- Vercel 检测到推送后会自动重新部署（约 1-2 分钟）

### 3. 测试

部署完成后：
- 访问网站首页
- 应该能看到所有文章，包括新添加的
- 新文章会排在最前面（如果按时间排序）

## 注意事项

### 性能影响

- **动态渲染**：每次请求都会重新生成页面
- 对于个人网站，性能影响很小
- 如果文章很多（>1000篇），可以考虑使用 ISR（增量静态再生）

### 如果以后需要优化

可以使用 ISR（增量静态再生）：
```typescript
export const revalidate = 60; // 每60秒重新生成一次
```

这样既能保持性能，又能定期更新内容。

## 完成

修改后，新文章会立即显示在首页，无需等待重新构建！
