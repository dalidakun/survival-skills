# GitHub 存储空间限制说明

## GitHub 仓库存储限制

### 1. 仓库大小限制

- **建议大小**：不超过 **1GB**
- **硬限制**：最大 **100GB**（但超过 1GB 会有性能问题）
- **警告阈值**：超过 **1GB** 时，GitHub 会发送警告邮件

### 2. 单个文件限制

- **警告阈值**：**50MB** - GitHub 会显示警告，但仍可提交
- **硬限制**：**100MB** - 超过此大小会被拒绝提交
- **我们的限制**：代码中已设置 **50MB** 限制，符合 GitHub 建议

### 3. Git LFS（大文件存储）

如果文件超过 50MB 或需要存储大量大文件：

- **免费账户**：
  - 存储空间：**1GB**
  - 带宽：**1GB/月**
- **付费账户**：
  - 存储空间：50GB 起（$5/月）
  - 带宽：50GB/月 起

## 对您的网站的影响

### 当前情况

- **文章文件**（`.md` 文件）：非常小，通常几 KB 到几十 KB
- **PDF 文件**：可能较大，但限制在 50MB 以内

### 存储空间估算

假设：
- 每篇文章：约 5KB
- 每个 PDF：平均 5MB（假设）
- 100 篇文章 + 100 个 PDF = 5MB + 500MB = **约 505MB**

**结论**：在合理使用范围内，不会超过 1GB 限制。

### 如果存储空间不足怎么办？

#### 方案 1：使用 Git LFS（适合少量大文件）

如果 PDF 文件超过 50MB 或接近 50MB：

```bash
# 安装 Git LFS
git lfs install

# 跟踪 PDF 文件
git lfs track "public/pdfs/*.pdf"

# 提交 .gitattributes
git add .gitattributes
git commit -m "Add Git LFS tracking for PDFs"
```

**限制**：免费账户只有 1GB 存储空间。

#### 方案 2：使用外部存储服务（推荐用于大量文件）

如果 PDF 文件很多或很大，建议使用：

1. **Cloudflare R2**（推荐）
   - 免费额度：10GB 存储 + 100 万次读取/月
   - 完全免费，无需信用卡

2. **Backblaze B2**
   - 免费额度：10GB 存储
   - 下载流量：每天 1GB 免费

3. **GitHub Releases**
   - 每个 Release 可以附加文件
   - 适合存储较大的文档

#### 方案 3：优化 PDF 文件

- 压缩 PDF 文件大小
- 使用在线工具减少文件体积
- 只上传必要的 PDF

## 建议

### 短期（当前方案）

✅ **继续使用 GitHub 存储**，因为：
- PDF 文件限制在 50MB 以内
- 通常不会超过 1GB 总限制
- 简单、免费、可靠

### 长期（如果文件增多）

如果 PDF 文件超过 50 个或总大小接近 1GB：

1. **监控仓库大小**：
   - 在 GitHub 仓库页面查看仓库大小
   - 定期检查是否接近 1GB

2. **考虑迁移到外部存储**：
   - 使用 Cloudflare R2 或 Backblaze B2
   - 修改代码，PDF 上传到云存储
   - 文章链接指向云存储的 URL

## 如何查看当前仓库大小

1. 访问 GitHub 仓库页面
2. 点击 "Insights" 标签
3. 查看 "Network" 或使用第三方工具

或者使用命令行：

```bash
git count-objects -vH
```

## 总结

- ✅ **当前方案可行**：50MB 限制 + GitHub 存储足够使用
- ⚠️ **注意监控**：如果 PDF 文件很多，注意仓库大小
- 🔄 **未来可扩展**：如果超过限制，可以迁移到云存储
