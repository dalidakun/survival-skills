# PDF 存储方案推荐

## 需求分析

- **文档数量**：几百个 PDF
- **每个文档大小**：约 15MB
- **总存储需求**：几百 × 15MB = 几 GB 到十几 GB
- **要求**：免费、长期可用链接、不影响网站查阅

## 推荐方案：Cloudflare R2

### 为什么选择 Cloudflare R2？

✅ **完全免费**：
- 10GB 存储空间（免费）
- 100 万次读取/月（免费）
- 无需信用卡
- 无隐藏费用

✅ **性能优秀**：
- 全球 CDN 加速
- 访问速度快
- 99.9% 可用性

✅ **永久链接**：
- 可以生成公开访问的 URL
- 链接长期有效
- 支持自定义域名

✅ **易于集成**：
- 提供 API
- 支持直接上传
- 与现有代码集成简单

### 存储空间估算

假设 300 个 PDF，每个 15MB：
- 总大小：300 × 15MB = 4.5GB
- **在 10GB 免费额度内** ✅

如果超过 10GB：
- 可以压缩 PDF 文件
- 或使用多个 Cloudflare 账户（每个 10GB）

## 其他备选方案

### 方案 2：Backblaze B2

- **免费额度**：10GB 存储
- **下载流量**：每天 1GB 免费
- **缺点**：下载流量限制较严格

### 方案 3：GitHub Releases

- **限制**：每个 Release 可以附加文件
- **缺点**：不适合管理几百个文件
- **不推荐**用于此场景

## 实施步骤

### 步骤 1：注册 Cloudflare 账户

1. 访问：https://dash.cloudflare.com/sign-up
2. 使用邮箱注册（完全免费）
3. 验证邮箱

### 步骤 2：创建 R2 存储桶

1. 登录 Cloudflare Dashboard
2. 点击左侧菜单 "R2"
3. 点击 "Create bucket"
4. 输入存储桶名称（如：`survival-skills-pdfs`）
5. 选择位置（建议选择离您最近的区域）
6. 点击 "Create bucket"

### 步骤 3：获取 API Token

1. 在 R2 页面，点击 "Manage R2 API Tokens"
2. 点击 "Create API Token"
3. 设置权限：
   - **权限**：Object Read & Write
   - **TTL**：永久（或设置较长时间）
4. 保存 Token（只显示一次，请妥善保存）

### 步骤 4：配置公开访问

1. 在存储桶设置中，启用 "Public Access"
2. 配置 CORS 规则（允许网站访问）
