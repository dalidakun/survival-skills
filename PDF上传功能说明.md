# PDF 上传功能说明

## 功能作用

**是的，这个功能就是把您本地的 PDF 文件上传到网站。**

### 工作流程：

1. **您上传 PDF**：在管理后台选择"上传 PDF"模式，选择本地 PDF 文件
2. **系统处理**：
   - 上传 PDF 文件到服务器
   - 生成一个可访问的链接（如：`/pdfs/文件名.pdf`）
   - 自动创建一篇文章，链接指向这个 PDF
3. **其他人可以访问**：
   - 文章会显示在首页
   - 点击文章标题，会打开 PDF 文件
   - 任何人都可以通过链接访问 PDF

## 文件存储位置

### 当前实现（有问题）

- **存储位置**：`public/pdfs/` 目录
- **访问方式**：通过 `/pdfs/文件名.pdf` 访问
- **问题**：在 Vercel 上，`public/` 目录是只读的，无法写入文件

### 在 Vercel 上的问题

由于 Vercel 使用只读文件系统，当前的 PDF 上传功能**在 Vercel 上无法正常工作**。

## 解决方案

需要修改为使用 GitHub API 来保存 PDF 文件，就像保存文章一样。

### 方案 1：修改为使用 GitHub API（推荐）

将 PDF 文件也通过 GitHub API 提交到仓库的 `public/pdfs/` 目录。

### 方案 2：使用外部存储服务

- 使用 GitHub 仓库直接存储（手动上传）
- 使用云存储服务（如 Cloudflare R2、Backblaze B2）

## 当前状态

- ✅ **本地开发**：可以正常上传 PDF
- ❌ **Vercel 部署**：无法上传（文件系统只读）

## 建议

如果您需要 PDF 上传功能在 Vercel 上工作，我可以修改代码，使用 GitHub API 来保存 PDF 文件。
